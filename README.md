## 🚀 About the project

This project started as a backend exercise, but evolved into a structured REST API.

The goal was not just to build a Todo List, but to design a clean and consistent API that could be used in real applications.

---

## ⚙️ What you can do

- retrieve lists and tasks through a REST API  
- navigate nested resources (`/lists/:listId/items`)  
- explore structured JSON responses  
- test the API in a public read-only environment  

The public demo runs in read-only mode to keep data consistent.

---

## 🧠 What I focused on

While building this project, I worked on:

- designing a clean REST API structure  
- organizing backend logic in a modular way  
- handling validation and errors consistently  
- managing data with SQLite  
- simulating production constraints (rate limiting, read-only mode)  

This is a backend-first project focused on structure and reliability.

---

## 🛠️ Tech Stack

- Node.js  
- Express  
- SQLite (`better-sqlite3`)  
- Vanilla JavaScript (frontend demo)  
- GitHub Actions (CI)
